失业之面试篇

By Lanveer; Published on 2019-08-27

  • Interview

面试经历

前言

  终于如愿以偿领取了例子大礼包，高兴之余，也要进行下一阶段的旅途了，于是更新简历，复习面试题等等就需要提上日程了，于是这篇文章就应运而生了，在这里我会记录每一次的面试，希望这篇文章越短越好，这样就说明没面试几家就找到了工作。

电话面试

  今天是8月27，晚上接到了电话面试，主要问到了

  1. es6的新特性
  2. js的原型链了解以及具体解说一下
  3. redux和dva的具体作用，状态和数据管理
  4. promis的介绍
  5. react的ref介绍
  6. js的继承相关
  7. 深拷贝和浅拷贝
  8. dva中的构成
    大概就是问了这些，当然这是做一个简单的沟通，记录下来，自己这段时间都没有怎么复习，回答的时候有一些力不从心，看来是时候准备一番了。后续有什么题目再进行下一步的更新

  今天是8月29，早上去了一家公司面试，在经历了紧张的等待之后，开始了面试，由于这位面试官不是前端，而是一位后端所以基本上都没有展开前端的知识点进行提问，都是围绕着项目在交谈。下面还是总结一下这次的主要问题

  1. 权限控制

  2. react的状态管理

  3. session在不同页面之间的共存

  4. 搭建项目所遇到的问题

  5. 对现在项目的看法

    下午又马不停蹄的赶往下一场面试，这场面是可谓是败得惨烈，第一轮还好，基本上都能答上来，到了第二轮，就彻底的歇菜了，很多东西都说不出来，尴尬的过了几分钟结束了面试，久久不能平静啊，自己知道这些缺陷已经很久了，却总是没有去弥补，到现在需要用到的时候就这个样子。 想想也不是滋味啊，不过一句老话，福兮祸之所伏，祸兮福之所倚，塞翁失马焉知非福，这一次的失利，却暴露了自己缺乏的东西。正好可以借此机会迷途知返，查漏补缺，把这些自己不会的东西再给补上来。

    让自己出过一次错就不要再次出错。第一面问的题目较多，也比较基础，所以就按照记忆，罗列一些，主要是二面自己的问题。

    一面：

  6. all， apply，bind，new的区别以及作用

  7. 抖节流的实现

  8. 拷贝浅拷贝的实现

  9. 型链的讲解

  10. js的事件循环

  11. rl到结果的事件

  12. ss相关的一些操作

  13. s类型判断

  14. 组的map和reduce有什么作用

  15. 阶组件
    二面：

  16. 绍作用域链

  17. 何判断js的类型（手写instance实现）

  18. eact新增的hook代表什么

  19. git rebase 和merge的区别

  20. flex相关知识

  21. 如何考虑封装一个组件

    反正经历了这一次面试，自己是真的要吸取经验教训了，不然很多好好的机会就会白白的流失了，跌倒过的地方就要避免再次跌倒。

  今天是8月30,今天真是充实的一天，面试了四个公司，虽然有两个是电话面试，但是也不影响，还是趁着有印象，赶紧把能够回忆起来的东西给记录下来，总而言之，今天也算是记录了很多之前不曾注意的问题。

一， 这是一家匹配度并不是很合适的公司，但是因为答应了hr，所以还是去试了一下，按照惯例，做了一套题，题目主要是围绕着基础来的，

  1. css 的优先级
  2. JS的数据类型
  3. 单描述闭包（保护私有变量，不受全局污染，这是下一场面试小哥帮我解答的）
  4. all，apply，bind，三者的区别（这里需要记录一下，后者是不会立即执行的，前两者会立即执行，这也是很重要的一个知识点）
  5. 述一下promise的工作原理
  6. 数作用域的一道题，谁调用，this就会指向谁，还有就是通过that=this做this的缓存
  7. 组去重（set方法是可以的，还有就是对象遍历方法，避免数组堆叠，和性能优化）
  8. 下的就是一些vue相关的操作，生命周期函数，以及computed，methods，watch，filter等等。还有一个$route和$router的区别，vues等等
  9. 有一些人性能优化，seo优化等等。

基本上面试也就是围绕着这些问题展开的，回去反观了一下，问题都是比较基础，遗憾的是自己没有使用vue，所以很多东西不太明白。不过也趁此机会，好好了解了一下。

二.这算是一家教育行业的公司，好的是有自己的产品，地点也比较方便，第一家面试完了之后，很快就赶到了这一家，其实在进门之前，接到了另外一个电话面试，暂时就放在下面描述。这家企业居然有两个面试官一起面试，很新鲜，还是按照老惯例，贴出我还能记住的题目。

  1. js的事件代理
  2. js的事件循环
  3. 移动端的适配，
  4. 处理跨域请求
  5. 深拷贝和浅拷贝
  6. flex布局
  7. css动画或者是transform
  8. react-router 如何传参数

三， 这是一个电话面试，感觉聊了挺久的，问的问题也特别的多，同时也掌握了一些平时都不注意的东西，不管结果好坏，还是记录一下。

  1. 简单介绍项目以及自我介绍是免不了的
  2. react的生命周期函数
  3. this.setstate为什么要加一个callback
  4. react中为什么需要加一个key
  5. html5的新特性
  6. cookie，session， localstorage，sessionstorage区别
  7. 如何实现定宽和不定宽高的垂直居中（flex实现，grid-兼容性，绝对定位，transform）
  8. 数组的去重做法
  9. webpack优化
  10. 父子组件之间如何通信以及调用方法（父组件调用子组件的方法是需要使用到ref）
  11. edux的使用（数据流的解释）
  12. 盒模型
  13. meta标签的作用
  14. link和import的区别
  15. 闭包的使用，以及简单的介绍
  16. diff算法

四，回家以后又经历了昨天约好的电话面试，感觉面试完了都不知道说了一些什么，记性没有忘性大。

  1. 项目介绍
  2. redux介绍
  3. http缓存
  4. react和vue angular的对比、
  5. virtualDOM的好处以及介绍
  6. 询问了node项目
  7. es6 新特性的产生

今天由于头晕脑胀，加上今天面试太多了，好些问题都搞混了，脑子不够用了，加上自身有很多事情影响了，所以状态很不好，先好好休息一下，看休息好了能不能想起来。

  今天是8月31，星期六，去了一个专场招聘,经历了两轮技术面试，也做了笔试题，收获良多，这里补充一下自己欠缺的知识体系，以及将遇到的题目做一个记录。

笔试题：

  1. 比较常见的，浏览器内核，html元素，数据类型，原型链,作用域以及常见的api

  2. css实现垂直居中以及定宽的flex布局

  3. 数组去重

  4. css 的优先级，创建节点原生js的考察，http请求状态吗的考察，get post的区别

  5. 严格模式和普通模式的代码执行，作用域的题目

  6. js事件流的执行顺序
    这些四笔试里面出现的知识点，具体的题目我没有整理，接下来记录一下面试中问到的题目

    一面：

  7. react hooks的理解以及引用场景

  8. https 和http的区别在哪里（这个需要好好看一下，仔细比对一下二者的不同）

  9. http缓存的使用

  10. 性能优化的地方（react方向）

  11. 前端工程化的看法（这一个比较新颖，需要下来好好斟酌一下）

  12. 考察了一个实际的业务场景（在线选座位）

  13. css3动画以及过度的使用

  14. redux的流程

  15. node在项目中的作用以及实际的应用场景

  16. 日常的开发流程，分支处理

  17. 对小程序的使用

  18. react里面组件的封装以及自己有没有封装一些组件之类的东西

  19. cdn的使用，以及DNS的了解

    这一轮面试持续的时间比较长，问的问题也是比较的多，有好些都想不起来了，需要好好的回忆一下，想好了再补充就是。

    二面：
    这议论是一个后端大佬的面试，主要的问题不在于讨论技术，而在于讨论平常的开发流程。

  20. 还是比较在平常的开发流程以及开发规范

  21. 浏览器的渲染原理的分析（构建dom和css 以及最后形成的render tree的一系列流程）

  22. 自己在项目中遇到的问题以及怎么解决（开发过程中出现的额问题，，前后端协同的时候遇到的困难）

  23. 页面装载的时候network中的loaded 和documentloaded 的差别是什么，

  24. 对项目的优化，业务上的和加载上的

  25. 自己的定位以及规划

    大概就是聊了这些，可能由于紧张的原因吧，有一部分的东西忘记了，等到后面想起来了再补充就是。

具体能够回忆起来的：
